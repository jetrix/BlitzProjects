BlitzScript3D Engine by Frankie Taylor

Welcome to my BlitzScript3D Demo. This is by far my fastest and most flexible Scripting Engine, blazing at 1/4th the speed of Blitz itself. Based on Mark Sibly - Blitz Compiler Code, BlitzScript3D Engine compiles a Scripts (*.bsc) with a Blitz3D-like Syntax to Pcode (*.bso) and executes on a Virtual Machine.

This help file should provide enough information to launch the Demo. Detailed help and information on the operation of the BlitzScript3D Engine is soon to follow. If additional assistance is required please contact me via email frankie_taylor@hotmail.com.

Included Files:

  • bscvm.exe or bscvm.bb - Main Demo Program
  • test.bsc - BlitzScript3D File
  • README.html - Demo Help

How to Launch the Demo

Execute bscvm.exe or bscvm.bb.

<Esc> - Stop Running Script
<R> - Reload and Run Script
<X> - Exit Demo

Currrently, the Script is executed with each screen update. This can be controlled, however, has been omited for now. So keep this in mind when editing the test script.

How to Edit the Test.bsc Script File

You can edit and save the test.bsc file, then reload and run the script. Your changes take effect instantly without recompiling Blitz. Currently, BlitzScript3D Compiler is limited and strict, but, advance functionality is to be added. Its syntax is very similar Blitz3D, however, it is missing many conveniences.

BlitzScript3D Supports:

Keywords:

let (optional)
if... then....
if... then....else

Note: BlitzScript3D is case sensitive. Currently all keywords, variables, and commands are lower case! This is subject to change in the future.

Operators and Expressions:

Binary +, -, *, /, <, >, <=, =>, =, <>
String @ or $+(string concenate), $* (string multiplier)
String Compare $<, $>, $<=, $=>, =, <>
Unary -,*,/,<<,>> have precedence over +, -,which have precendence over <, >, <=, =>, =, <>
Parenthesis can be used to group subexpressions.

Variables:

BlitzScript3D is virtually typeless handling values like a string or floats (integer values append a .0 ie: 4.0, 23.0, 10000.0). Currently, all variables are public in scope, as functions are not supported yet. Variables labels can begin with A-Z or underscore '_' character and followed with alphanumeric or underscore characters.

BlitzScriptVar

BlitzScript3D also features a special global array 'BlitzScriptVar' that can be assigned values and accessed in both script and application using the following commands:

BlitzScriptVarSet(index,value)
BlitzScriptVarGet(index)

Examples

blitzscriptVarSet(1,"Hello World")

blitzscriptVarSet(25,450.0)

print(blitzscriptVarGet(2)

Blitz Commands

Over 471 Blitz3D Commands are Supported, however, there are some slight differences.

1) Parameter(s) must be enclosed within Parenthesis. Missing Parenthesis will result in error.

Example: positionentity(camera,0,0,-3,0)

2) BlitzScript3D accounts for ALL Parameters of the Blitz Command, thus, you cannot omit a Default Parameters. Absense of any Parameters will result in error.

Example: cone=createcone(8,1,0)

BlitzScript3D Commands

BlitzScript3D Commands are Hardcoded BlitzScript3D Functions. The same rules apply to them as with Blitz Commands.. There are few at this time...more to be added. A list of BlitzScript3D Commands will be developed as these commands are added.

BlitzScript3D Does NOT Support:

Arrays (to be implemented)

Types (to be determined)

Conditional Loops (to be implemented)

Advance Conditions (to be implemented)

Functions (to be implemented)

Includes (to be implemented)

At the moment, the Scripting System can only parse and execute one BlitzScript3D Code File at a time (multiple file support is to be determined, until then, consolidate needed scripts in one file). The BlitzScript3D Virtual Machine processing is currently limited to 256 scripts.

Tabs

At the moment the BlitzScript3D Compiler will not recognize Tabs and results a in syntax error. Recommendation: Replace them with specified number of spaces. I'm working on a fix.

Remarks/Comments

BlitzScript3D will allow Single and Multi-line Remarks/Comments. Support is Coming Soon.

How Does It Work?

Scripts are stored in the *.bsc file. Upon loading the Engine will search for a compiled version of the file *.bso. If a compiled version is found its loaded, otherwise, its compiled then loaded.

Once loaded. The Virtual Machine executes each script one after another per update loop. This behavior will be changed in the future.
